llm402.ai just added image generation to its L402-gated API, which means your agent can now generate images for as little as 21 sats — no account, no API key, no prepaid balance. Just a lightning wallet and a prompt.

## What llm402.ai Offers

[llm402.ai](https://llm402.ai) is an OpenAI-compatible API gateway where every request is gated by an L402 payment. Instead of signing up and loading credits, your agent makes a request, receives a `402 Payment Required` with a lightning invoice, pays it, and retries — all automatically.

Image generation lands at the standard `/v1/images/generations` endpoint with a range of models:

| Model          | Price    |
| -------------- | -------- |
| FLUX.1-schnell | 21 sats  |
| FLUX.2-pro     | 63 sats  |
| FLUX.2-max     | 147 sats |
| Ideogram 3.0   | 126 sats |
| GPT-5-image    | 105 sats |

The cheapest option (FLUX.1-schnell at 21 sats) is fast enough for real-time workflows. The most capable option (GPT-5-image) produces dramatically more detailed output for about five times the price.

## Generating an Image with One Command

The [Alby payments skill](https://github.com/getAlby/payments-skill) handles L402 automatically. Install it once:

```bash
npx skills add getAlby/payments-skill
```

Then your agent can generate an image with a single call — the skill handles the 402 handshake, payment, and retry:

```bash
npx -y @getalby/cli fetch -X POST \
  -H '{"Content-Type":"application/json"}' \
  -b '{
    "model": "black-forest-labs/FLUX.1-schnell",
    "prompt": "A vending machine dispensing AI-generated artwork for lightning bolt coins, each slot showing a different image style, pixel art",
    "width": 1200,
    "height": 630
  }' \
  https://llm402.ai/v1/images/generations
```

The image comes back as a base64-encoded data URI in the response body. No separate download step, no polling — just a synchronous result.

## How It Works Under the Hood

The L402 flow is straightforward:

1. **Initial request** — agent POSTs to `/v1/images/generations`
2. **402 response** — server replies with a `WWW-Authenticate: L402` header containing a macaroon and a BOLT11 lightning invoice
3. **Payment** — agent pays the invoice using its NWC-connected wallet
4. **Authenticated retry** — agent re-sends the request with `Authorization: L402 <macaroon>:<preimage>`
5. **Image returned** — server verifies payment and returns the generated image

The Alby payments skill wraps steps 2–4 into a single function call, so agents don't need to implement the protocol manually.

## Using It in a TypeScript Agent

```typescript
import { fetch402 } from "@getalby/payments-skill";

const response = await fetch402("https://llm402.ai/v1/images/generations", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "black-forest-labs/FLUX.1-schnell",
    prompt: "your prompt here",
    width: 1024,
    height: 1024,
  }),
});

const { data } = await response.json();
const imageUrl = data[0].url; // base64 data URI
```

Switch `model` to `openai/gpt-5-image` for higher quality — same code, different cost.
