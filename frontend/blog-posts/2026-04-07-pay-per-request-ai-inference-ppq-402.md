AI agents need access to AI APIs — but managing prepaid credits for each service is friction that doesn't belong in an autonomous workflow. PPQ.ai now supports native 402 payments via the MPP (Machine Payments Protocol), so your agent can pay for exactly what it uses, one request at a time.

## The Old Way: Credit Topups

Until recently, using PPQ.ai in an automated workflow meant maintaining a credit balance — loading funds upfront and hoping you'd budgeted correctly. For agents running in production, this worked well enough with NWC auto topups: you could set a threshold and have the wallet automatically top up from a lightning wallet when the balance ran low.

That approach still works great. But it has overhead: you're managing a separate account balance, monitoring thresholds, and pre-funding access to an API you might only call occasionally.

## The New Way: 402 Pay-Per-Request

PPQ.ai now supports the [402 Payment Required](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/402) flow via MPP — Machine Payments Protocol. When your agent makes a request, PPQ.ai responds with a `402` and a payment request. Your agent pays, then retries the original request automatically.

No balance to manage. No topups to configure. Your agent pays exactly what the request costs — nothing more.

## Unified 402 Fetch in the Alby Payments Skill

The [Alby bitcoin payments skill](https://github.com/getAlby/payments-skill) added a unified `fetch402` function that handles all three major 402 payment protocols in a single call:

- **L402** — the original LSAT-based HTTP 402 protocol
- **x402** — Coinbase's 402 variant
- **MPP** — Machine Payments Protocol, used by PPQ.ai

Install the skill:

```bash
npx skills add getAlby/payments-skill
```

Once installed, your agent can call any 402-gated API without knowing which protocol it uses — the skill detects and handles it automatically.

## Example: Generating an Image from PPQ.ai

Here's what a real agent session looks like. Given the prompt:

> "Generate an image from PPQ.ai using the 402 option and alby. The prompt: 'a superhero bee shouting 402 is awesome'"

The agent used the payments skill's CLI to make the request:

```bash
npx -y @getalby/cli fetch \
  -u https://api.ppq.ai/v1/images/generations \
  -m POST \
  -H '{"Content-Type":"application/json"}' \
  -b '{"model":"gpt-image-1","prompt":"a superhero bee shouting '\''402 is awesome'\''","n":1,"size":"1024x1024","quality":"low"}' \
  --max-amount 0
```

The `--max-amount 0` flag tells the CLI to approve any payment amount automatically. The result:

```json
{
  "created": 1775576898,
  "data": [
    {
      "url": "https://api.ppq.ai/v1/media/gen_zp-zdKLuKQfAjO7m/0?sig=...",
      "content_type": "image/png"
    }
  ],
  "model": "gpt-image-1",
  "cost": 0.023
}
```

The agent paid $0.023 — exactly the cost of that one image, settled over lightning. No account needed. No balance to top up. No API key to rotate.

## Example: Generating a Video from PPQ.ai

The same 402 flow works for video generation. Given the prompt:

> "Generate a short video of the superhero bee paying to pass through an API gate, using the 402 option and alby"

The agent made the request using `kling-2.1-master`:

```bash
npx -y @getalby/cli fetch \
  -u https://api.ppq.ai/v1/videos \
  -m POST \
  -H '{"Content-Type":"application/json"}' \
  -b '{"model":"kling-2.1-master","prompt":"A cartoon superhero bee wearing a cape flies up to a glowing hexagonal API gate on a dark navy background. The bee drops a tiny golden lightning bolt coin into a glowing slot on the gate. The gate flashes bright cyan and swings open. The bee zooms through triumphantly. Flat minimal animation style, no text, seamless loop.","duration":"5","aspect_ratio":"16:9","quality":"720p"}' \
  --max-amount 0
```

The response comes back as a pending job with a `status_url` to poll. Once complete:

```json
{
  "id": "gen_y5eM1UCxQT7bNVUd",
  "model": "kling-2.1-master",
  "status": "completed",
  "data": {
    "url": "https://api.ppq.ai/v1/media/gen_y5eM1UCxQT7bNVUd/0?sig=...",
    "content_type": "video/mp4"
  },
  "cost": 0.92
}
```

$0.92 for a 5-second 720p video, paid over lightning with no account or API key:
